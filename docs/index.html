How to read these charts
  • Different algorithms, different knobs. SereneDB = IVF (nprobe), Qdrant = HNSW (hnsw_ef) — not the same unit, so compare the recall-vs-QPS curve, not points at equal effort.
  • "Best per band" = highest QPS at that recall@10 across every config measured, ties broken by lower p50. Bands are 0.05-wide up to 0.95 recall, then 0.01-wide above it.
  • QPS is closed-loop at 32 concurrent clients, same protocol for both engines.
  • Build-time/index-size panels are per-engine, not head-to-head (different structures and knobs). SereneDB is pinned at nlist_factor=2 (RaBitQ only ran at nlist_factor=1, so it's absent here). The build-time bar stacks CREATE INDEX under its VACUUM (COMPACT_TABLE) merge from one build; index size shows compact vs. no-compact and includes the base-table columnstore copy (Qdrant's is index-only).
